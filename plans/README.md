## Dependency notes

- **003 requires 002**: 003 changes (or, in the recommended Option A,
  documents) the whitespace contract; the characterization tests from 002 make
  that change visible and prevent silent regressions.
- **005 / 006 lean on 002**: their behavior probes reuse 002's input set and
  helpers.
- **003 ↔ 006 overlap**: 003 is the *tactical* fix (document the
  builder-vs-classifier contract; zero breakage). 006 is the *strategic* one
  (unify the two parsers so the divergence cannot recur). If 006 is adopted, it
  supersedes 003's manual alignment — do 003 first anyway (cheap, ships clarity
  now) and note the overlap in the 006 PR.
- **005 ↔ 006 ordering**: if both proceed to implementation, settle the mode
  contract (005) before unifying the core (006) so the single core honors it.
- **F4 (LEGADO) is handled inside 005**, not as a standalone fix — deciding
  whether to deprecate or define LEGADO is exactly 005's design question, so a
  premature deprecation elsewhere would pre-empt it.

## Findings considered and rejected

- **pip-audit reported 11 vulnerabilities** (pip, pytest, pygments, idna,
  urllib3, starlette): all in the local dev/CI virtualenv, none in the
  library's runtime dependencies (the core has zero). starlette is a transitive
  of the optional `fastapi` extra. CI already runs separate lockfile/dev-deps
  audits. Not a package finding — no plan.
- **`_sanitizar_mensaje` bare-numeric bypass** (`exceptions.py:11`): the PII
  regex requires a `-[0-9kK]` suffix, so an over-length all-digits value (e.g.
  `ErrorLongitudRut` on `"1234567890"`) is logged unsanitized. Marginal: only
  triggers on malformed, over-length, non-RUT numeric input — not real PII —
  and the value is already invalid. Note-and-reject unless future work in
  `exceptions.py` makes the one-line regex hardening free.
